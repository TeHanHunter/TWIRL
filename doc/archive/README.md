benchmarks, and exploratory reasoning that are useful for provenance but are
not current operating instructions.

2. [`doc/twirl_plan.md`](../twirl_plan.md) for the forward-looking survey plan.
3. [`doc/twirl_progress_log.md`](../twirl_progress_log.md) for current dated
   execution state.
4. Active operational runbooks such as the
   [A2v1 protocol](../a2v1_production_protocol.md).

Archived snapshots:

  earlier broad brainstorming and referee-risk notes.

The legacy QLP-standard transition and canonical-detrending sector protocols
remain at their historical paths with explicit supersession notices, because
old reports link to them. They must not be used for new A2v1 production.

Git history still contains older collaboration and career-strategy prose that
was removed from the live plan. This cleanup does not rewrite history; any
future history purge would be a separate, explicitly authorized operation.
