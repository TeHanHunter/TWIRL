- `2026-05-06`: First-pass design choices, all forced by what the WD 1856 smoke test exposed:
    - Search **all three apertures** (`DET_FLUX_SML`, `DET_FLUX`, `DET_FLUX_LAG`). Aperture-of-record varies per CCD/contamination вЂ” S56 cam4/ccd1 passes via `DET_FLUX`, S59 cam4/ccd4 only passes via `DET_FLUX_SML` (medium and large are scattered-light dominated on that CCD).

    - `n_periods = 200,000`, `n_peaks = 20`. Compute is cheap; keeping more peaks lets the cross-aperture step rescue weak signals that aren't a single aperture's rank-1.
    - **Upper-tail-only 5-Пѓ clip** (asymmetric). A symmetric clip would erase real WD transits вЂ” at `MAD-RMS=0.10`, the `57%` deep WD 1856 dip is a `5.7-Пѓ` negative excursion. Scattered-light wings are positive only.
    - 200 s cadence Г— 8 min transit means transits are sampled at ~`2.4` cadences each, with random alignment between cadence center and transit mid. Effective measured depth is `~30-50%` of truth across many transits, regardless of aperture; never trust BLS depth as a candidate filter.

    - **Pulsator / ellipsoidal binary**: smooth sinusoidal phase fold with no flat baseline; identical odd/even depths; agree=3. Example: `TIC 320323992` (`T=15.7`, `P=0.121 d`, `SDE 1980-2700` across all 3 apertures, odd-even О” `0.000`).

    - **Real WD eclipsing binary**: V-shape at WD-relevant period, similar depth in SML+MED, odd-even consistent. Example: `TIC 27766711` (`T=15.8`, `P=0.388 d ~ 9.3 h`).
- `2026-05-07`: **Updated physics knowledge** in [twirl_plan.md Design Drivers](twirl_plan.md): for WDs, `R_companion >= R_host` is the *common* case (`R_WD ~ 0.013 R_sun ~ 1 R_jup`, so any planet larger than Mars-sized has `R_p > R_WD`). Transit duration is set by the larger body, scaling with `R_companion + R_WD` rather than `R_host` alone. Implication: the current BLS duration grid (`3, 5, 8, 12, 20 min`) is sized for WD-only minute-scale transits and **truncates WD+M-dwarf post-CE binaries** (durations 30-90 min). Future widening to `~3 hr` is on the table for the next pass.
- `2026-05-07`: Decision on **vetting strategy**: do *not* train our own ML model yet, do *not* use Astronet/LEO-vetter as-is. Both are tuned for main-sequence host priors that fail catastrophically on WDs (LEO-vetter's stellar-density check forbids WD-fast transits at any plausible host). With `1` confirmed WD planet (WD 1856) the positive-class is too small for deep learning. Plan: build a TWIRL-specific heuristic vetter on engineered features (aperture-depth ladder slope, out-of-transit baseline flatness vs in-transit scatter, predicted-vs-observed duration from Gaia parallax, multi-sector persistence) вЂ” interpretable, additive, debuggable. Defer ML until ~Year 2 with `~10k` labeled FPs and a small random forest on the same engineered features.

Then re-run filter pass with these features to demote pulsators / contaminator-ladders / duration-violators.
